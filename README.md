# rtm-cli-go

Command-line client for the
[Remember The Milk API](https://www.rememberthemilk.com/services/api/).

This module is mostly hand-written. Two subdirectories are
generated by [rtm-gen-go](https://github.com/morozov/rtm-gen-go)
and committed as-is:

- `internal/rtm/` — the RTM API client (stdlib-only).
- `internal/commands/` — the cobra command tree, exposed through
  a `Register(root, provider)` function that `cmd/rtm/main.go`
  calls.

Everything else is hand-written: the binary entry point, the
root cobra command, persistent flags, credential sourcing, and
anything else that isn't an RTM API binding.

## Build and install

```sh
go install github.com/morozov/rtm-cli-go/cmd/rtm@latest
```

Or from a clone:

```sh
go build -o rtm ./cmd/rtm
./rtm --help
```

## Run

```sh
rtm --key $RTM_API_KEY --secret $RTM_API_SECRET --token $RTM_AUTH_TOKEN \
    <service> <method> [flags]
```

`--key` and `--secret` are required for every invocation.
`--token` is required for methods that need a logged-in user
(most writes and some reads).

The CLI mirrors the RTM service hierarchy:

```sh
rtm reflection get-methods
rtm auth check-token
rtm lists get-list
rtm tasks add --name "Ship it" --list-id 123
rtm tasks notes add --list-id 1 --taskseries-id 2 --task-id 3 --note-title "..."
```

Each invocation makes one RTM call and writes the raw JSON
response body to stdout.

## Regenerate

The generator is `github.com/morozov/rtm-gen-go`, pinned via the
`tool` directive in `go.mod`. Regeneration reads a local spec
cache at `./spec.json` (gitignored).

Populate the cache first — either by hand-copying an existing
dump, or by fetching live with RTM credentials:

```sh
go tool rtm-gen client -key $RTM_API_KEY -secret $RTM_API_SECRET -out /dev/null >/dev/null 2>&1 || true
# Or just place a pre-existing api.json at ./spec.json.
```

Then:

```sh
go generate ./...
go mod tidy
go build ./cmd/rtm
```

`go generate` invokes `rtm-gen client` and `rtm-gen cli`
indirectly through `//go:generate` directives at
`internal/rtm/gen.go` and `internal/commands/gen.go`. Those
files are hand-written and carry no generated header; every
other file in the two `internal/` packages starts with
`// Code generated by rtm-gen; DO NOT EDIT.`

### Pre-publish note

`rtm-gen-go` is not yet published to a module proxy. Until it
is, `go.mod` carries a local `replace` directive pointing at the
sibling checkout `../rtm-gen-go`. The `replace` is only used for
regeneration (`go tool rtm-gen ...`); `go install
.../cmd/rtm@latest` still works against the committed output.

## Layout

```
rtm-cli-go/
├── cmd/rtm/main.go          (hand-written)
├── internal/
│   ├── rtm/
│   │   ├── gen.go           (hand-written; carries //go:generate)
│   │   ├── client.go        (generated)
│   │   └── <service>.go     (generated, one per RTM service)
│   └── commands/
│       ├── gen.go           (hand-written; carries //go:generate)
│       ├── register.go      (generated)
│       └── <service>.go     (generated, one per RTM service)
├── go.mod
├── go.sum
└── README.md
```
